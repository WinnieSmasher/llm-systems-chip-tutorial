# 00C. 学习协议

这份教程不是收藏夹，也不是百科。它按“读者能不能交付一个小成果”来判断学习是否完成。

每章默认遵守下面的学习协议。

## 1. 读前先写问题

开始读一章之前，先写 3 个问题。问题越具体越好。

不要写：

```text
学习 vLLM
学习 CUDA
学习 RAG
```

要写：

```text
TTFT 和 TPOT 分别对应推理链路里的哪一段？
ZLUDA 为什么不能等同于昇腾 CANN？
chunk size 变大时，RAG 召回为什么可能变差？
```

## 2. 每章只认一种小产出

读完一章，至少留下一个东西：

| 章节类型 | 最小产出 |
| --- | --- |
| 概念章节 | 一张对照表或边界图 |
| 工程章节 | 一个能跑的最小脚本 |
| 评测章节 | 一份固定输入和结果记录 |
| 源码章节 | 一页源码阅读笔记 |
| 论文章节 | 一页 paper note |

没有产出，就当作只是看过，不算学完。

## 3. 记录失败样例

教程里的实验不要求一次成功。失败样例更有用，因为它能告诉你系统边界在哪里。

建议记录：

```text
我想验证什么：
我运行了什么：
结果哪里不符合预期：
可能原因：
下一步怎么缩小问题：
```

例子：

- vLLM 服务能启动，但 TTFT 很慢。
- RAG 检索到了相似文本，但没有检索到正确证据。
- ONNX 导出成功，但后端转换失败。
- LoRA smoke test 能跑，但输出质量没有稳定改善。

## 4. 一章的完成标准

学完一章时，至少能做到三件事：

1. 用自己的话解释本章核心概念。
2. 指出一个常见误区。
3. 拿出一个小产出作为证据。

例如学完 CUDA / ZLUDA / CANN，不能只说“都和硬件有关”。至少要能解释：

- CUDA 是 NVIDIA 原生生态。
- ZLUDA 是 CUDA compatibility layer。
- CANN 是昇腾 NPU 原生软件栈。
- ONNX / OM 是模型格式和部署链路里的文件，不是硬件生态本身。

## 5. 笔记模板

每章可以用这个模板：

```text
章节：
读前问题：
核心概念：
我跑了什么 / 画了什么：
失败样例：
我现在能解释什么：
还没懂的问题：
下一章：
```

如果你把这个模板认真填完，这个仓库就不只是 README，而会变成自己的学习档案。
